Build the ROC static C++ library for robust optimization

This file describes how to use Microsoft Visual Studio to:
  • build a static library from the ROC source code.

Building Your Own Robust Optimization library with ROC source code

Note:
The information below applies to the Visual C++ 2012 multi-threaded STL library.

  1. Start Microsoft Visual Studio 2012.

  2. The first step is to build the ROClib.sln solution.
    From the File menu, select New->, and then Project....

    The New Project dialog box appears.

    • In the Project Types pane, select Visual C++ Projects.
    • In the Templates pane, select the Win64 Project icon.
    • Fill in the project name (ROClib).
    • If necessary, correct the location of the project (to <MYAPPDIR>)
    • Click OK

    3. When the Win64 Application Wizard appears...

    • Click on Application Settings.
    • Select Console Application as Application type.
    • Make sure that Empty project is checked in Additional Options.
    • Click Finish.

    This creates a solution, ROClib, with a single project, ROClib. You can see the contents of the solution by selecting Solution Explorer in the View menu.

  3. Now you must add your source file to the project. From the Project menu, choose Add Existing Item...
    • Move to the folder <ROCDIR> and select *.cpp and *.h.
    • Click Open.

  4. Next, you have to set some options so that the project knows where to find the CPLEX and Boost.

    From the Project menu, choose Properties.
    The ROClib Property Pages dialog box appears.

    In the Configuration drop-down list, select Release.

    Select Configuration Properties tree.

    • Select General:
      • Select Project Defaults, in the Configuration Type field, choose Static library(.lib).

    Select C/C++ in the Configuration Properties tree.

    • Select General:
      • In the Additional Include Directories field, add the directories:
        • <CPLEXDIR>\include.
        • <CONCERTDIR>\include.
        • <BoostDir>\include.
      • For Debug Information Format, choose Disabled (/Zd).

    • Select Preprocessor:
      • Add IL_STD, _USE_CPLEX_ to the Preprocessor Definitions field. This defines the macro IL_STD and _USE_CPLEX_ which are needed to use the STL.

    Click OK to close the test Property Pages dialog box.

  5. Next, you have to set the default project configuration.

    From the Build menu, select Configuration Manager...

    • Select Release in the Active Solution Configuration drop-down list.
    • Click Close.

  6. Finally, to build the project, from the Build menu, select Build Solution

Remark:

Here, we need both the Boost and CPLEX library installed.
